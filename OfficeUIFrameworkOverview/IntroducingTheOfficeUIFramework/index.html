Introducing the OfficeUI Framework

The OfficeUI Framework is a rich command presentation system that provides a modern alternative to the classic HTML controls of thaditional web applications.

Views

The primary UI components of the OfficeUI Framework (the Ribbon, the Context Menu, the Grid and the Modal) are differentiated structurally through Views. The Framework itself support 4 different View. The Ribbon view, the ContextPopup view, the Grid View and the Modal view.

The Ribbon View

The UI of the Ribbon if the primary feature of the OfficeUI framework and provides a starting point for ceating next-generation web applications using a very intuitive interface.

The ribbon is a command bar that exposes the features of an application through a series of tabs at the top of a website. It it similar in functionality and appearance to the Microsoft Office 2013 Fluent UI. The ribbon provides an intuitive counterpoint to the trial-and-error process of discovery that is quite common on standard web applications. Optimized for efficiency and discoverability, the ribbon facilitates finding, understanding and using commands with a minimum of mouse clicks and keystrokes through a system of standard controls.

The ContextPopup View

The ContextPopup View provides a rich ContextMenu that can be used in combination with the Ribbon to expose additional commands on the Ribbon, but it can also be used as a stand alone menu somewhere within the website. The ContextPopup view provides additional features such as sub menu items, disabled menu items, icons and seperators.

The Grid View

The Grid view provides a way to display a collection of items in a grid representation. In that Grid, you can select items, execute actions based on a selected row, ...

The Modal View

The Modal view provides a way to draw the user's attention on something important. When the Modal view is opened, it also blocks any input to other elements which are on the page.
