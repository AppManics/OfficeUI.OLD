Downloading the package

The topics contained in this section explore the various release packages.

When you want to use the OfficeUI framework, it's recommended to download it from our official Github account.
For every release version there are 2 packages avaialble:

  • Development package
  • Release package

Development package

The development package is a fully working package but it isn't minified and contains various elements that enables the user of this package to track down bugs in this application.

It's recommended to always start with this package. Since the development package and the release package is guaranteed to have the same code-base, just with some optimazations, there's no need to worry that your application will not be working anymore when moving from a development package to a release package in the future (as long as you keep the same version).

Production pacakge

The production package is a fully optimized package which has been created from the development package. By fully optimized, we mean that logging has been removed from this package and it's also minified to keep the file size as low as possible.

In an ideal word, your development environment (for example: Visual Studio) should always include a development package. When your application is released, on the server, just make sure to place a production version of the package with the same package version.
