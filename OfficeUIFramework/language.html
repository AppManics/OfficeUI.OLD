Declaring Commands and Controls with OfficeUI Markup

The OfficeUI framework can be rendered using HTML to declaratively implement the appearance of an OfficeUI application.

The OfficeUI site has been built up so that it's possible to render fully functional views from any of the following source files:

  • Extensible Markup Language (XML)
  • JavaScript Object Notation (JSon)
If you don't like any of those formats and you would like to have full control over the rendered HTML, you can always create the HTML yourself.

Note  It is recommended to either use JSon or XML to render the OffceUI views. However, it's possible to render it specifying the HTML, the smallest mistake can produce some very strange results which you defintely want to avoid at all the times.

Since every View has it's own specific syntax, view the topic in the View which you want to render.
