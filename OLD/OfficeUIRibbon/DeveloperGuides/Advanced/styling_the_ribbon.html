Styling the ribbon

The ribbon has a couple of themes that can be used which are Blue, Green, Orange or Purple.
In this section you'll learn how to apply them or how to create your own.

Using one of the built-in themes

As described in the section Creating your first Ribbon application, you have a couple of stylesheets that you need to include in your application. 

<link rel="stylesheet" type="text/css" href="Resources/Stylesheets/OfficeUI.Core.css" />
<link rel="stylesheet" type="text/css" href="Resources/Stylesheets/Plugins/Ribbon/OfficeUI.Ribbon.css" />
<link rel="stylesheet" type="text/css" 
href="Resources/Stylesheets/Plugins/Ribbon/Styles/OfficeUI.Ribbon.Blue.css" />
The entire theming of the Ribbon is hidden in the styles, you might notice that the last css that is included is Resources/Stylesheets/Plugins/Ribbon/Styles/OfficeUI.Ribbon.Blue.css

This means that the Ribbon will be rendered in Blue. The supported stylesheets for the Ribbon are:

  • OfficeUI.Ribbon.Blue.css
  • OfficeUI.Ribbon.Green.css
  • OfficeUI.Ribbon.Orange.css
  • OfficeUI.Ribbon.Purple.css

Creating your own themes

When you not satisfied with one of the built-in themes, you can always create your own one in a few simple steps:

Step 1: Create a new stylesheet for your theme

When you want to create a new theme, you need to create a new stylesheet. In order to maintain the readability and the locations of the file, the best-practice is to name the file "OfficeUI.Ribbon.{Theme}.css" and save it in the directory where the other themes for the Ribbon are being placed.

The default is: Resources/Stylesheets/Plugins/Ribbon/Styles/

Step 3: Provide all the necessary elements in the css file

When you create a new theme, you can either choose to start from an existing one of a new one. It is recommended to start from an existing one, so inside your newly created file, place the following contents: 

#OfficeUI li[role=tab].application { background-color: #2b579a; color: #FFF; border: 1px solid #2b579a !important; }
#OfficeUI li[role=tab]:not(.application):hover { color: #2b579a; }
#OfficeUI ul[role=tablist] li.active[role=tab] { color: #2b579a; }
#OfficeUI .icon.active { background-color: #a3bde3; }

#OfficeUI a:not(.OfficeUI_disabled) .bigicon:not(.active):hover, 
#OfficeUI a:not(.OfficeUI_disabled) .smallicon:not(.active):hover { background-color: #d5e1f2; }

Inside this stylesheet you'll find some classes and properties. The table below gives an explanation of all those.

Class Description
#OfficeUI li[role=tab].application Provides the class that styles an application tab. This is normally the tab on the left side.
#OfficeUI li[role=tab]:not(.application):hover Defines the text color of tab when hovering on it (not the application tab).
#OfficeUI ul[role=tablist] li.active[role=tab] Provides the text color of an active tab element.
#OfficeUI .icon.active Provides the background color of icon when it's active (when the mouse button is pressed on an icon).
#OfficeUI a:not(.OfficeUI_disabled) .bigicon:not(.active):hover

#OfficeUI a:not(.OfficeUI_disabled) .smallicon:not(.active):hover		 Provides the background color of an icon when hovering on it.
