Managing the package dependencies

As mentioned in the introduction there are a couple of dependencies on which the project does rely.
When you download the package, thoses dependencies comes packages with the download itself, but it's not necessary to use those dependencies, you can choose to update the dependencies to a newer version or to older versions if you want to try to use it on older versions that the recommended ones.

The table below gives an overview about the dependencies and where there are used.

Dependency Optional Description
jQuery No The OfficeUI framework needs this file to be present at all times.
jQuery UI Yes This file is used to render components using animations and is not needed when you don't use menu items.
Font awesome Yes This file is used to render components using animations and is not needed when you don't use menu items.

Dependency location

Every dependency is present in a special folder. It's not mandatory to use this folders, but when you change them, make sure to change the path where the files are being called, this usually is your HTML file.

The table below gives an overview of the defautl locationo of the dependencies.

Dependency Folder
jQuery ~/Resources/Scripts
jQuery UI ~/Resources/Scripts
Font awesome ~/Resources/Stylesheets
