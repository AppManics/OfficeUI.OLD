Running the package on your local computer

This section gives additional information on how to run the OfficeUI framework on your local computer.

When you've downloaded your package from the server, in the root of the package, you'll find a file named index.html
You might want to run the file at once, but unfortunately, this won't work.

The OfficeUI framework uses Json files in combination with templating files (*.tmpl) to render the complex HTML elements. By default, webbrowsers doesn't allow ajax calls (which are used to load the data) on a local filesystem, that's why you need to setup your own environment.

Setting up a local webserver

This tutorial will use Microsoft IIS as webserver to setup an environment in which the OfficeUI framework will be able to operate. You're free to choose whatever webserver you like.

Note: The screenshots in this example are taken from an IIS instance on a Windows 8.1 Pro machine. The interface might be different on other versions.

  1. Create a new website in IIS:

    Create a new wbsite in IIS with the following parameters:

    1. Site Name: Choose your desired sitename here.
    2. Application pool: Free to choose.
    3. Physical path: Select the root of the OfficeUI framework folder here.
    4. Host Name: Any hostname you like (for example: www.officeui.com)

  2. Defining MIME Types:

    3. Open your created website in Internet Explorer and inside the Features view, navigate to "MIME Types".
    In here you need to define 2 MIME Types:

    .tmpl: text/plain
    .json: application/json

    Note: When you're running the application as a ASP.NET application a web.config is included which sets the MIME-Type already, so in that case, it's not needed to setup the mime types in your webserver.

  3. Changing your host file

    4. Open the file C:\Windows\System32\Drivers\etc\hosts if you're running on Windows and add the following line:

    1. 127.0.0.1 www.officeui.com

    Note: When you're not running Windows, you should add this line to the equivalent file for the Operating System on which you're working.

You should now be able to access the website using the url 'http://www.officeui.com'
