Understanding the interface

The Ribbon is splitted in several parts (tabs, groups, action groups and actions). In this guide, you'll learn what those different aspects are and where they come into play.

When you take a look at the Ribbon in the first place, you might notice that everything in the Ribbon has it's own dedicated place. The following elements are standard on a ribbon:

  • Tabs
  • Groups
  • Action Groups
  • Actions

Tabs

The tabs on the Ribbon is a way to group different actions into a menu item.
In the image below, the tabs are identified by a red border. In this particular case, we have 5 tabs defined (File, Home, Send / Receive, Folder and View).

Note:  The first tab (File) is colored in blue. This is because this tab is an application tab, but we'll see more about this in Understanding Ribbon Controls

Groups

Each Tab element can have another separation for the controls that are displayed on the Ribbon. This is called a group.
A group can be identified by the seperator (vertical line) which appears on the right side of it, or at the legend that's on the bottom of the group.
In the image below, the groups are identified by a red border, in this particular case, we have 3 groups (New, Delete and Respond).

Action groups

Inside a group, we can have different action groups. An icon group is a mechanism that allows us to place some actions in the same area. An action group contains a set of actions. All the action groups are displayed horizontally in the Ribbon.
In the image below, the icon groups are identified by a red border.

Actions

An action on the ribbon can represent various controls that allow's the user to execute something on the website.
The following actions are supported:

  • Small icon (with menu)
  • Big icon (with menu)
